# Laravel Docker Setup

這次學習在Docker建立Laravel的開發環境，方便日後上生產環境(Production)

參考: 用 15 分鐘快速打造 Laravel 開發環境 , Laradock

假設本身沒有Laravel Project，從第一步開始

目標目錄架構, 這樣做方便日後多個項目, Multiple Projects

https://laradock.io/getting-started/

├── laradock
└── project1
└── project2
└── ...

首先clone一份laradock.git

git clone https://github.com/Laradock/laradock.git

Cloning into 'laradock'...

進入laradock folder, 復制env-example 進行自定義

cd laradock
cp env-example .env
vim .env

另外為了避免80 443 port 已被使用，所以更改到8000 4430

​```bash NGINX_HOST_HTTP_PORT=8000 NGINX_HOST_HTTPS_PORT=4430


進入容器 `nginx` 和 `mysql`, 會自動生成`network`, `container`等項目

```bash
docker-compose up -d nginx mysql

然後進入剛剛生成的laradock_workspace_1, 日後的所有php的操作都是在這裡進行. 根據.env 中APP_CODE_PATH_CONTAINER，所以一開始的路徑在/var/www

docker-compose exec workspace bash

root@93e56517c403:/var/www#

在建立文件夾project1, 並生成Laravel項目

mkdir project1
cd project1
composer create-project laravel/laravel --prefer-dist .

如果這時看本地文件的話，會看到多了一個project1的文件夾，因為docker是與本地同步的

假設domain是project1.test, 增加一個.conf檔案

cd ../laradock/
cp -r nginx/sites/laravel.conf.example nginx/sites/project1.test.conf

把修改server_name和root

vim nginx/sites/project1.test.conf

server_name project1.test;
root /var/www/project1/public;

更新一下nginx的設定

docker-compose exec nginx nginx -s reload

注意：要在本機的hosts檔添加127.0.0.1 project1.test

現在上project1.test:8000，應該可以看到Laravel的畫面

還有些後續操作, 如果我們要migrate, 會有Connection refused, 因為我們未正確設定.env中的DB_HOST

php artisan migrate

 Illuminate\Database\QueryException

  SQLSTATE[HY000] [2002] Connection refused (SQL: select * from information_schema.tables where table_schema = laravel and table_name = migrations and table_type = 'BASE TABLE')
  ...

將DB_HOST由127.0.0.1改成mysql, 另外填入預設密碼

Connecting Laravel Project with Host Mysql

vim .env

DB_HOST=mysql
DB_PASSWORD=root

再試一次migrate, 這次成功連接到mysql, 但有錯誤: Unknown database 'laravel', 原因是一開始在mysql沒有一個database叫laravel

php artisan migrate

  Illuminate\Database\QueryException

  SQLSTATE[HY000] [1049] Unknown database 'laravel' (SQL: select * from information_schema.tables where table_schema = laravel and table_name = migrations and table_type = 'BASE TABLE')

現在我們要進入mysql的docker container

docker-compose exec mysql bash

root@52107ec0d277:/#

登入mysql, 預設帳號密碼都是root

mysql -u root -p

Enter password:
...
mysql>

生成database

create database laravel;

Query OK, 1 row affected (0.01 sec)

現在再試一次migrate, 成功!

docker-compose exec workspace bash
cd project1
php artisan migrate

Migration table created successfully.
Migrating: 2014_10_12_000000_create_users_table
Migrated:  2014_10_12_000000_create_users_table (0.04 seconds)
Migrating: 2019_08_19_000000_create_failed_jobs_table
Migrated:  2019_08_19_000000_create_failed_jobs_table (0.01 seconds)

嘗試一下開始auth, 看看能否正常運作

https://stackoverflow.com/a/34546836/5588637

https://laravel.com/docs/6.x/frontend

cd project1
composer require laravel/ui
php artisan ui bootstrap --auth
php artisan migrate
npm install && npm run dev

成功!!
